Tailwind CSS: Building Beautiful and Responsive Web Designs

by Natu

July 13, 2024

What is Tailwind CSS?

Tailwind is a utility framework of CSS that enables us to style the components using the markup classes directly. Just like bootstrap, the primary distinction is that bootstrap is a component-based framework, while tailwind is more like a utility-based framework.

Tailwind offers additional opportunities to style the components in a flexible manner. With Tailwind, you can also utilize features like media queries or pseudo elements like hover, active, etc. straight into your markup element classes without coding for any additional CSS in another file or such.

What is the difference between component-based and utility-based frameworks?

Component-based frameworks like Bootstrap, Foundation, and Materialize are meant to make it simpler to construct and maintain user interfaces by employing reusable components. Frameworks often provide a collection of preset modular components that may be stylized and changed to match the demands of a project.

While utility-based frameworks like Tailwind and Bootstrap (with utility classes) are more like classes, which offer power to style components at lower levels,. They are single-purpose classes that may be combined into style components. These frameworks concentrate on allowing developers comprehensive control over style without having to create individual CSS for every component. 

Why use Tailwind CSS?

Think about writing lengthy styles like “margin-top: 1rem” and comparing it with "mt-1." Which one appears more intriguing and simpler to write? Tailwind enables us to write the styles inside the classes of a tag, so there’s no need to create another CSS file or style tags until you have to alter anything in a group.

Example:

Without Tailwind

                    
<h1>CodroidHub</h1>
h1 { color: red; }

With Tailwind

                    
<h1 class="text-red-600">CodroidHub</h1>
Alternatively, we can paste the same code with the style tags in an external CSS file and import that into our index.html.

Tailwind made creating code and adding media queries simple. While without tailwind, it makes the code more hefty since it also contains the classes and styles that we are not utilizing in the page. One of the major characteristics of Tailwind CSS is that it only adds those classes to the stylesheet that are utilized in the page and disregards the classes that are not.

How to utilize Tailwind CSS?

Before digging into how to utilize and configure Tailwind CSS, it’s vital that you read its documentation. Every framework in the world has its own documentation, which developers or designers can study and act on appropriately. Let’s get started!

Many beginners, like myself when I first started to understand Tailwind CSS, become confused with the structure of it. But it’s not as complex as it looks. Tailwind can be added to your project in a variety of ways. But we are going to use the CDN method today. 

Installing Tailwind using CDN

Most of the novices would utilize Tailwind CSS on a standard organized page where there’s an index.html file with style.css and script.js.

Adding tailwind CDN script

Let's add a script link to our index.html section below: 

<script src="https://cdn.tailwindcss.com"></script>
If you open the supplied URL in the script, you’ll notice a minimum of JavaScript that is used to load the files. The essential feautres that tailwind requires

Creating tailwind.config.js

In the next step we’ll paste the following script in the head section of the index.html 

                      
  <script>
    tailwind.config = {
      theme: {
        extend: {
          colors: {
            clifford: '#da373d',
          }
        }
      }
    }
  <script>
Alternatively, we can create a new file named tailwind.config.js, copy the same code into it, and import it into our index.html.

The tailwind.config is used to customize our own color, and much more, and use them globally in our markups. You can read it in more detail in the Tailwind Documentation.

Managing custom styles

Tailwind does most of our work, but if there is something custom that we want to add, We can use the following code to add our own custom CSS classes and id styles: 

                      
  <style type="text/tailwindcss">
    @layer utilities {
      .content-auto {
        content-visibility: auto;
      }
    }
  <style>
Alternatively, we can paste the same code with the style tags in an external CSS file and import that into our index.html.

Conslusion

Tailwind CSS represents a significant shift in the way developers approach styling in modern web development. Its utility-first approach encourages the creation of highly reusable, modular styles that can be easily customized and extended. By leveraging a predefined set of classes that directly map to CSS properties, Tailwind minimizes the need for custom CSS, streamlining the development process and enhancing consistency across a project.

This methodology not only accelerates the design workflow but also improves maintainability by reducing the amount of CSS code and fostering a more systematic approach to styling. As Tailwind continues to evolve and gain adoption, it stands as a powerful tool for creating visually appealing, responsive designs with greater efficiency and less overhead.
